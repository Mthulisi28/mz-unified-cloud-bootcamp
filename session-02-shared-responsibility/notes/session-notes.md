\# ◆ SESSION 2 TEACHING NOTES



\---



\# ◆ OPENING SCRIPT



Say:



“Cloud is powerful — but power without responsibility creates risk.”



\---



\# ◆ SHARED RESPONSIBILITY EXPLANATION



Explain simply:



AWS handles:

\- Physical infrastructure



Customers handle:

\- Everything inside AWS



\---



\# ◆ SIMPLE ANALOGY



Apartment example:



\- AWS = landlord

\- Customer = tenant



Landlord secures building  

Tenant secures belongings



\---



\# ◆ IAM EXPLANATION



IAM = access control system.



Ask:

\- Who can log in?

\- What can they do?



\---



\# ◆ SECURITY GROUPS



Explain:

\- Firewall for EC2

\- Controls traffic flow



\---



\# ◆ COMMON RISKS



Teach clearly:

\- Public access mistakes

\- Weak passwords

\- Missing MFA

\- Over-permissions



\---



\# ◆ KEY MESSAGE



“Most cloud failures are caused by misconfiguration, not AWS failures.”
