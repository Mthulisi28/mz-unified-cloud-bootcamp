\# AWS Cloud Bootcamp



\## Session 2: Shared Responsibility \& Security



\---



\## OBJECTIVES

\- Understand Shared Responsibility Model

\- Learn IAM basics

\- Understand Security Groups

\- Identify cloud risks



\---



\## SHARED RESPONSIBILITY MODEL

Security is shared between:

\- AWS

\- Customer



\---



\## AWS RESPONSIBILITY

AWS manages:

\- Physical infrastructure

\- Datacenters

\- Hardware

\- Networking



\---



\## CUSTOMER RESPONSIBILITY

Customers manage:

\- Data

\- IAM users

\- Security configuration

\- Applications



\---



\## CORE PRINCIPLE

“AWS secures the cloud. You secure what you put in the cloud.”



\---



\## IAM (IDENTITY \& ACCESS MANAGEMENT)

IAM controls:

\- Who can access AWS

\- What they can do



\---



\## SECURITY GROUPS

Security Groups = virtual firewall for EC2.



Controls:

\- Inbound traffic

\- Outbound traffic



\---



\## CLOUD RISKS

\- Public access exposure

\- Weak passwords

\- No MFA

\- Over-permissioned users



\---



\## SUMMARY

\- Security is shared

\- Misconfiguration causes most cloud issues
