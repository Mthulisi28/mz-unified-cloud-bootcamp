\# AWS Cloud Foundations Bootcamp



\## Session 1: Cloud Computing Fundamentals



\---



\## OBJECTIVES

\- Understand cloud computing

\- Understand AWS global infrastructure

\- Learn Regions and Availability Zones



\---



\## WHAT IS CLOUD COMPUTING

Cloud computing = on-demand computing resources over the internet.



Key idea:

\- No physical servers owned by users

\- Pay-as-you-go model

\- Scalable infrastructure



\---



\## WHY CLOUD EXISTS

\- Cost efficiency

\- Global reach

\- High availability

\- Faster deployment



\---



\## AWS GLOBAL INFRASTRUCTURE

AWS is built using:

\- Regions

\- Availability Zones

\- Edge Locations



\---



\## REGIONS

A Region is a geographic location where AWS operates.



Example:

\- Cape Town

\- London

\- Virginia



\---



\## AVAILABILITY ZONES

AZs are isolated data centers inside a Region.



Purpose:

\- High availability

\- Fault tolerance

\- Disaster recovery



\---



\## SUMMARY

\- Cloud = rented computing power

\- Regions = geographic areas

\- AZs = redundant data centers
