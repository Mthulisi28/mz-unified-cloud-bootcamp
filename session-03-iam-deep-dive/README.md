\# ◆ SESSION 3 — IAM DEEP DIVE



\## Objective

Build deep understanding of AWS Identity and Access Management (IAM) and how permissions control cloud security at scale.



\---



\# ◆ WHAT YOU WILL LEARN



\- IAM Users, Groups, Roles

\- IAM Policies (what they actually control)

\- Principle of Least Privilege

\- Temporary credentials concept

\- Real-world access control thinking



\---



\# ◆ WHY THIS MATTERS



Most cloud breaches happen because of:

\- Over-permissioned IAM users

\- Poor role design

\- Misconfigured policies



IAM is the foundation of cloud security.



\---



\# ◆ CORE PRINCIPLE



“Never give more access than needed.”



\---



\# ◆ KEY CONCEPTS



\## IAM USER

A person or system accessing AWS.



\## IAM GROUP

A collection of users with shared permissions.



\## IAM ROLE

Temporary identity used by services or applications.



\## IAM POLICY

A JSON document that defines permissions.



\---



\# ◆ LEARNING OUTCOME



Students will understand:

\- How AWS controls access

\- Why permissions must be restricted

\- How roles improve security
