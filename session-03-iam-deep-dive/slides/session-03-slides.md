\# SESSION 3 — IAM DEEP DIVE



\## OBJECTIVES

\- Understand IAM structure

\- Learn access control fundamentals



\---



\## IAM USERS

Identities for individuals or systems.



\---



\## IAM GROUPS

Collections of users with shared permissions.



\---



\## IAM ROLES

Temporary access identities for services.



\---



\## IAM POLICIES

Define what actions are allowed or denied.



\---



\## PRINCIPLE OF LEAST PRIVILEGE

Give only required permissions.



\---



\## SUMMARY

IAM is the foundation of AWS security.
