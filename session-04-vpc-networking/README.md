\# ◆ SESSION 4 — VPC \& NETWORKING FUNDAMENTALS



\## Objective

Understand how AWS networking works using VPCs, subnets, and routing concepts.



\---



\# ◆ WHAT YOU WILL LEARN



\- What a VPC is

\- Public vs Private Subnets

\- Internet Gateway basics

\- Route Tables concept

\- How AWS networking is structured



\---



\# ◆ WHY THIS MATTERS



Every AWS system depends on networking.



If networking is wrong:

\- Applications break

\- Security fails

\- Systems become unreachable



\---



\# ◆ CORE CONCEPT



A VPC (Virtual Private Cloud) is your isolated network inside AWS.



\---



\# ◆ KEY COMPONENTS



\## VPC

Your private network in AWS.



\## SUBNETS

Divisions inside a VPC:

\- Public Subnet → internet access

\- Private Subnet → no direct internet access



\## INTERNET GATEWAY

Allows communication between VPC and internet.



\## ROUTE TABLE

Defines traffic flow rules inside the network.



\---



\# ◆ LEARNING OUTCOME



Students will understand:

\- How AWS networks are built

\- Why segmentation is important

\- How traffic flows in cloud systems
